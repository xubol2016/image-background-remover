### 2. BgRemover 项目接入 Google OAuth 登录功能

**项目背景**：
- BgRemover 是一个基于 Next.js 14 + Cloudflare Pages 的在线图片背景移除工具
- 当前已实现核心功能：图片上传、背景移除、结果下载
- 需要添加用户认证系统，以便后续实现用户历史记录、使用限制等功能

**技术选型分析**：
| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| 纯前端 Implicit Flow | 简单，无后端依赖 | 安全性较低，Token 暴露 | ❌ 不适合 |
| 后端 Authorization Code Flow | 安全性高 | 需要 Secret，流程复杂 | ❌ 过度设计 |
| **Google Identity Services (GIS)** | 安全、简单、无需 Secret | 仅适用于 Google 账号 | ✅ 选定 |

选择 GIS 的原因：
1. 项目已使用 Cloudflare Pages Functions，可以处理后端验证
2. GIS 直接返回 ID Token（JWT），后端用公钥验证即可，无需 Secret
3. 用户体验好，使用 Google 官方登录按钮，信任度高

**具体实施步骤**：

#### 2.1 依赖安装
```bash
npm install jose
```
选择 `jose` 而非 `jsonwebtoken` 的原因：
- `jose` 支持 Web Crypto API，更适合 Edge Runtime（Cloudflare Workers）
- 体积更小，无原生依赖
- 支持远程 JWKS 自动获取和缓存

#### 2.2 后端 API 开发

**文件 1：`functions/api/auth/google.ts`**
- 功能：接收前端传来的 Google ID Token，验证签名，创建会话
- 验证逻辑：
  1. 从 Google 公钥端点 (`https://www.googleapis.com/oauth2/v3/certs`) 获取 JWKS
  2. 使用 `jwtVerify` 验证 Token 签名、issuer、audience
  3. 提取用户基本信息（id, email, name, picture）
  4. 生成随机 session token (UUID)
  5. 将用户数据存入 Cloudflare KV，设置 7 天过期
  6. 返回 session token 给前端

**文件 2：`functions/api/auth/me.ts`**
- 功能：获取当前登录用户信息
- 流程：
  1. 从 Authorization Header 提取 Bearer Token
  2. 查询 KV 中 `session:${token}` 对应的用户数据
  3. 返回用户信息或 401 错误

**文件 3：`functions/api/auth/logout.ts`**
- 功能：用户登出
- 流程：
  1. 从 Header 提取 token
  2. 从 KV 中删除对应 session
  3. 返回成功状态

#### 2.3 前端认证系统开发

**文件 4：`app/context/AuthContext.tsx`**
- 实现 React Context API 管理全局认证状态
- 核心功能：
  - `login(credential)`：调用后端 API，存储 token 到 localStorage
  - `logout()`：调用后端 API，清除 localStorage
  - `user`：当前用户信息（id, email, name, picture）
  - `isLoading`：认证状态初始化中的加载状态
- 持久化：使用 localStorage 存储 session token，页面刷新后自动恢复登录态

**文件 5：`app/components/GoogleLoginButton.tsx`**
- 功能：渲染 Google 官方登录按钮
- 实现细节：
  - 动态加载 Google Identity Services 脚本 (`https://accounts.google.com/gsi/client`)
  - 使用 `window.google.accounts.id.initialize()` 初始化
  - 使用 `window.google.accounts.id.renderButton()` 渲染按钮
  - 按钮样式：outline 主题、pill 形状、200px 宽度
  - 登录成功后调用 `AuthContext.login()` 处理凭证

**文件 6：`app/components/UserAvatar.tsx`**
- 功能：已登录状态下显示用户头像和下拉菜单
- UI 设计：
  - 圆形头像 + 用户名（移动端隐藏）+ 下拉箭头
  - 点击展开下拉菜单：显示用户全名、邮箱、退出登录按钮
  - 点击外部区域自动关闭下拉菜单
  - 退出按钮带红色高亮和图标

**文件 7：`app/layout.tsx`**

### 学到的知识和经验

1. **Google Identity Services 与传统 OAuth 2.0 的区别**
   - GIS 是 Google 推出的新一代登录方案，取代了旧的 Google Sign-In
   - 核心变化：直接返回 ID Token（JWT），而不是 Authorization Code
   - 优势：流程更简单，无需后端换取 Token，安全性仍然有保障（JWT 签名验证）
   - 适用场景：只需要用户身份信息的应用，不需要调用 Google API

2. **Cloudflare Pages Functions 的 KV 绑定**
   - 本地开发时通过 `.env.local` 使用模拟 KV
   - 生产环境需要在 `wrangler.toml` 中绑定 KV namespace
   - KV 是最终一致性，不适合需要强一致性的场景（但会话存储没问题）
   - KV 有读写限制（每天 10 万次读取），但对于个人项目足够

3. **Next.js 静态导出（Static Export）的限制与技巧**
   - `output: 'export'` 模式下，所有页面在构建时生成 HTML
   - API Routes 不会被导出，但 Cloudflare Pages Functions 可以在 `functions/` 目录中定义
   - 环境变量需要显式暴露在 `next.config.js` 中才能在前端使用
   - `images.unoptimized: true` 是静态导出的必需配置

4. **JWT 验证的最佳实践**
   - 始终使用成熟的库（如 `jose`），不要自己实现 JWT 解析
   - 验证签名、issuer、audience、过期时间缺一不可
   - 使用 JWKS 端点动态获取公钥，不要硬编码
   - 注意 issuer 可能有多个值（`https://accounts.google.com` 和 `accounts.google.com`）

5. **Edge Runtime 的兼容性**
   - Cloudflare Workers 使用 V8 Isolate，不是 Node.js
   - 某些 Node.js 内置模块不可用（如 `crypto` 需要改用 Web Crypto API）
   - `jose` 库完全兼容 Edge Runtime，是 JWT 验证的首选
